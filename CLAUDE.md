# Ergsn 프로젝트

## 배포

이 프로젝트의 배포 대상은 다음 저장소입니다:

- **Remote**: `https://github.com/ceodon/ergsn.git`
- **Branch**: `main`

"배포해줘" 요청이 오면 변경사항을 커밋하고 위 저장소의 `main` 브랜치로 푸시합니다.

### 커밋 정보

- `user.name`: `ceodon`
- `user.email`: `ceodon@gmail.com`

### 인증

푸시는 HTTPS + Personal Access Token을 사용합니다. 토큰은 저장소에 저장하지 않고,
푸시할 때마다 사용자에게 요청합니다. 사용 후 폐기(Revoke)를 안내할 것.

### 제외 항목

`.claude/` 디렉터리는 커밋하지 않습니다.

## 세션 핸드오프

사용자가 "다음 세션에서 진행한다" / "내일 이어서 하자" / "다음에 계속" 류의 표현으로
세션 종료 의사를 밝히면, 다음 세션이 바로 이어받을 수 있도록 핸드오프 문서를 작성합니다.

### 핸드오프 파일

- 위치: `.claude/handoff/YYYY-MM-DD-<slug>.md` (폴더가 없으면 생성)
- 파일명 예: `.claude/handoff/2026-04-17-chatbot-optimization.md`
- 같은 날 여러 건이면 뒤에 `-v2`, `-v3` 로 분기

### 포함할 내용

1. **날짜 & 세션 요약** — 오늘 한 일을 2–4 줄로 압축
2. **현재 상태** — 브랜치, 마지막 커밋 해시, 배포 여부, 미커밋 변경 유무
3. **진행 중인 작업** — 방금 중단된 지점(파일·라인·다음 할 일)
4. **결정된 사항** — 사용자가 확정한 방향/규칙 (추측 금지, 대화에 근거)
5. **미해결 질문 / 위험** — 다음 세션에서 확인 필요한 항목
6. **관련 메모리 포인터** — `memory/` 에 관련 메모리가 있으면 파일명 나열

### 메모리 동기화

핸드오프 작성 시, 다음 세션에도 유용한 `feedback` / `project` / `reference` 정보가
있으면 `memory/` 에 저장(또는 기존 항목 업데이트)하고 핸드오프에 링크를 남깁니다.
핸드오프는 "이번 세션의 스냅샷", 메모리는 "시간이 흘러도 유효한 규칙"으로 구분.

### 제외

`.claude/` 디렉터리는 커밋하지 않으므로, 핸드오프 문서는 로컬 전용입니다.
공유가 필요하면 사용자가 명시적으로 요청할 때만 다른 위치로 복사합니다.
