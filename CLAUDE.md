# Ergsn 프로젝트

## 배포

이 프로젝트의 배포 대상은 다음 저장소입니다:

- **Remote**: `git@github.com:ceodon/ergsn.git` (SSH)
- **Branch**: `main`

"배포해줘" 요청이 오면 변경사항을 커밋하고 위 저장소의 `main` 브랜치로 푸시합니다.

### 커밋 정보

- `user.name`: `ceodon`
- `user.email`: `ceodon@gmail.com`

### 인증

SSH 키 기반 자동 인증을 사용합니다. 사용자 개입 없이 바로 푸시하세요.

- 키 위치: `~/.ssh/id_ed25519` (ed25519, 2026-04-17 생성)
- GitHub 등록: `ceodon` 계정의 Authentication Key
- PAT은 더 이상 요구하지 않습니다. 혹시 SSH 인증이 실패하면 키 만료·삭제 가능성을
  사용자에게 알리고, PAT 폴백 전환은 사용자 승인 후 진행합니다.

### 제외 항목

`.claude/` 디렉터리는 커밋하지 않습니다.

## 세션 핸드오프

사용자가 "다음 세션에서 진행한다" / "내일 이어서 하자" / "다음에 계속" 류의 표현으로
세션 종료 의사를 밝히면, 다음 세션이 바로 이어받을 수 있도록 핸드오프를 갱신합니다.

### 단일 파일 방식

- 위치: `.claude/nextsession.md` (폴더가 없으면 생성)
- **날짜·slug 파일 만들지 말 것.** 항상 이 한 파일을 덮어쓰기(overwrite)로 갱신.
- 이전 내용은 보존하지 않음 — 다음 세션이 이 파일만 읽고 바로 이어받음.
- 같은 날 여러 번 세션 종료 신호가 와도 동일한 `nextsession.md` 를 다시 덮어쓴다.

### 포함할 내용

1. **최종 갱신 시각** — 파일 맨 위에 ISO 날짜·시간 한 줄 (예: `Updated: 2026-04-17`)
2. **세션 요약** — 이번에 한 일을 2–4 줄로 압축
3. **현재 상태** — 브랜치, 마지막 커밋 해시, 배포 여부, 미커밋 변경 유무
4. **진행 중인 작업** — 방금 중단된 지점(파일·라인·다음 할 일)
5. **결정된 사항** — 사용자가 확정한 방향/규칙 (추측 금지, 대화에 근거)
6. **미해결 질문 / 위험** — 다음 세션에서 확인 필요한 항목
7. **관련 메모리 포인터** — `memory/` 에 관련 메모리가 있으면 파일명 나열

### 메모리 동기화

핸드오프 갱신 시, 다음 세션에도 유용한 `feedback` / `project` / `reference` 정보가
있으면 `memory/` 에 저장(또는 기존 항목 업데이트)하고 `nextsession.md` 에 링크를 남깁니다.
`nextsession.md` 는 "이번 세션의 스냅샷(휘발성)", 메모리는 "시간이 흘러도 유효한 규칙(영구)" 로 구분.

### 제외

`.claude/` 디렉터리는 커밋하지 않으므로, `nextsession.md` 는 로컬 전용입니다.
공유가 필요하면 사용자가 명시적으로 요청할 때만 다른 위치로 복사합니다.
